# Билет 25: Escape-последовательности и r-строки

## Escape-последовательности

Внутри строки `\` начинает спец-символ:

| Запись | Значение |
|--------|----------|
| `\\` | сам обратный слеш |
| `\'` | апостроф |
| `\"` | кавычка |
| `\n` | новая строка (line feed) |
| `\t` | табуляция |
| `\r` | возврат каретки |
| `\v` | вертикальная табуляция |
| `\b` | backspace |
| `\0` | нулевой символ |

```python
print("a\tb")        # a    b
print("line1\nline2")# line1 / line2
print("кавычка: \"") # кавычка: "
print("слеш: \\")    # слеш: \
print('It\'s ok')    # It's ok
```

(!) `len` считает escape **как один символ**:

```python
print(len("a\nb"))   # 3 — \n это один символ
print(len("a\\b"))   # 3 — \\ это один символ (слеш)
```

## Проблема: пути и регулярки

В путях Windows и регулярках слешей много — приходится экранировать:

```python
path = "C:\\Users\\name\\new"   # неудобно
# "C:\new" — \n стал бы переводом строки!
```

## r-строки (raw strings)

Префикс `r` отключает обработку escape — `\` остаётся буквальным:

```python
print(r"C:\Users\name\new")   # C:\Users\name\new  — \n не перенос!
print(r"a\tb")                # a\tb — табуляции нет
print(len(r"a\nb"))           # 4 — символы a, \, n, b
```

### Мотивация использования r-строк

1. **Пути файловой системы** (особенно Windows):

```python
path = r"C:\temp\report.txt"
```

2. **Регулярные выражения** — чтобы `\d`, `\w`, `\b` не конфликтовали:

```python
import re
re.findall(r"\d+", "a12b34")   # ['12', '34'] — без r пришлось бы \\d+
```

## Тонкости r-строк

- r-строка **не может заканчиваться** на нечётное число `\`:

```python
r"C:\path\"     # SyntaxError — последний \ "экранирует" кавычку
r"C:\path"      # OK
"C:\\path\\"    # альтернатива без r
```

- r-строка не меняет тип — это обычная `str`, просто иначе записанная:

```python
print(r"a\nb" == "a\\nb")   # True
```

## Итоги

- escape: `\\ \' \" \n \t \r \v \b`; каждый — **один** символ для `len`;
- `r"..."` — raw: `\` буквальный, escape не обрабатывается;
- мотивация: пути Windows и регулярные выражения;
- r-строка не может оканчиваться нечётным числом `\`;
- `r"a\nb"` это та же `str`, что `"a\\nb"`.
