# Билет 4: id(), type, isinstance(), оператор is

## Зачем

Для проверки трёх характеристик объекта (идентичность, тип, значение) есть встроенные инструменты.

## Функция id()

`id(obj)` возвращает **целое число** — уникальный идентификатор объекта, постоянный за время его жизни.

```python
x = object()
print(id(x))     # напр. 140703...
```

**Смысл в CPython:** `id()` возвращает **адрес объекта в памяти** (CPython — эталонная реализация на C). Это деталь реализации: в других реализациях (PyPy, Jython) `id()` может вычисляться иначе. Гарантируется только: id уникален и постоянен, пока объект жив.

```python
a = []
b = []
print(id(a) == id(b))   # False — разные объекты
print(id(a) == id(a))   # True
```

## Объект type

`type` — встроенный объект (метакласс). `type(obj)` возвращает **класс** объекта.

```python
print(type(42))       # <class 'int'>
print(type("hi"))     # <class 'str'>
print(type([]))       # <class 'list'>
print(type(type))     # <class 'type'> — type сам объект type
```

## Функция isinstance()

`isinstance(obj, cls)` — проверка принадлежности типу **с учётом наследования** (в отличие от `type(obj) is cls`).

```python
print(isinstance(True, int))     # True! bool наследник int
print(type(True) is int)         # False — точное совпадение типа

# Можно передать кортеж типов
print(isinstance(3.0, (int, float)))   # True
```

> Предпочитайте `isinstance`, а не `type(x) == ...`: он корректно работает с наследниками.

## Оператор is

`is` сравнивает **идентичность** (по сути `id(a) == id(b)`), а `==` сравнивает **значение**.

```python
a = [1, 2, 3]
b = [1, 2, 3]
print(a == b)    # True  — равны по значению
print(a is b)    # False — разные объекты

c = a
print(a is c)    # True  — один объект
```

**Правило:** с `None`, `True`, `False` сравнивают через `is` (это синглтоны):

```python
if x is None:    # правильно
    ...
```

## Кеширование малых int

CPython кеширует маленькие целые от **-5 до 256** — для них `is` может неожиданно дать `True`. Это деталь реализации, на неё **нельзя полагаться**:

```python
a = 256; b = 256
print(a is b)    # True  (кеш)
a = 257; b = 257
print(a is b)    # False (обычно; зависит от контекста)
```

## Итоги

| Инструмент | Что проверяет |
|-----------|----------------|
| `id(x)` | идентичность (в CPython — адрес в памяти) |
| `type(x)` | точный тип |
| `isinstance(x, T)` | тип с учётом наследования |
| `is` | идентичность (тот же объект) |
| `==` | равенство значений |

- `id()` в CPython = адрес в памяти (деталь реализации);
- `is` ↔ идентичность, `==` ↔ значение;
- `None`/`True`/`False` сравнивать через `is`;
- `isinstance(True, int) == True`;
- кэш малых int (−5…256) — не полагаться.
