# Билет 1: Типизация в Python

## Динамическая и статическая типизация

Вопрос «когда проверяется тип» — это вопрос **момента связывания типа с именем/значением**.

| | Статическая | Динамическая |
|---|---|---|
| Когда известен тип | на этапе **компиляции** | на этапе **выполнения** |
| Пример языков | C, C++, Java, Go | Python, Ruby, JavaScript |
| Тип привязан к | **переменной** (имени) | **объекту** (значению) |

В Python **имя не имеет типа** — тип есть только у объекта, на который имя ссылается. Поэтому одно и то же имя может в разные моменты ссылаться на объекты разных типов:

```python
x = 42          # сейчас x ссылается на int
print(type(x))  # <class 'int'>

x = "hello"     # теперь то же имя ссылается на str — это ОК
print(type(x))  # <class 'str'>
```

> В статически типизированном языке (C, Java) такое переприсваивание с другим типом — ошибка компиляции.

## Строгая и нестрогая типизация

Этот вопрос — про **неявные приведения типов**.

- **Строгая (сильная) типизация** — язык не делает «магических» неявных преобразований несовместимых типов.
- **Нестрогая (слабая) типизация** — язык охотно приводит типы сам (например, `"3" + 4` даёт что-то).

Python — **строго типизированный**: несовместимые типы не складываются молча.

```python
print(2 + 3)        # 5    — оба int
print("2" + "3")    # '23' — оба str, это конкатенация
print(2 + "3")      # TypeError: unsupported operand type(s) for +: 'int' and 'str'
```

Сравните с JavaScript (`2 + "3" === "23"`) — там слабая типизация.

## Python = динамическая + строгая

Это две **независимые** оси. Python сочетает:

- **динамическую** типизацию — тип проверяется во время выполнения;
- **строгую** типизацию — неявных приведений несовместимых типов нет.

```python
def add(a, b):
    return a + b

add(1, 2)        # 3      — работает
add("a", "b")    # 'ab'   — работает (утиная типизация)
add(1, "b")      # TypeError во время выполнения — строгость
```

Аннотации типов (`x: int = 5`) в Python **не меняют** этого: интерпретатор их не проверяет, они для людей и для внешних анализаторов (mypy, pyright).

## Итоги

| Ось | Python |
|-----|--------|
| статическая / динамическая | **динамическая** (тип у объекта, проверка в рантайме) |
| строгая / нестрогая | **строгая** (нет неявных приведений) |
