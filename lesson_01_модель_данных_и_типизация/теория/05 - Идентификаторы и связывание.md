# Билет 5: Идентификаторы, связывание, ссылочная модель

## Идентификаторы

**Идентификатор** — имя для объекта (переменной, функции, класса, модуля).

Правила валидного идентификатора:
- состоит из букв, цифр и `_`;
- **не может начинаться с цифры**;
- регистрозависим (`name` ≠ `Name`);
- не может совпадать с **ключевым словом** (`class`, `for`, `def`, …);
- с Python 3 разрешены Unicode-буквы (`имя = 5` — валидно, но не рекомендуется).

```python
my_var = 1      # ✅
_hidden = 2     # ✅
var2 = 3        # ✅
2var = 4        # ❌ SyntaxError — начинается с цифры
my-var = 5      # ❌ дефис недопустим
class = 6       # ❌ ключевое слово
```

## Оператор = и операция связывания

`=` в Python — **не присваивание значения в ячейку**, как в C, а **связывание (binding)** имени с объектом. Имя становится ссылкой на объект.

```python
x = [1, 2, 3]   # имя x связывается с объектом-списком
```

## Различные формы связывания

Связывание имени происходит не только через `=`:

```python
x = 5                  # обычное присваивание
a, b = 1, 2            # распаковка
a = b = 0              # цепочка
x += 1                 # составное присваивание
for i in range(3):     # i связывается на каждой итерации
    ...
def f(): ...           # имя f связывается с функцией
import math            # имя math связывается с модулем
with open(p) as file:  # file связывается
class C: ...           # имя C
[y for y in data]      # y связывается внутри включения
```

## Ссылочная модель памяти

Ключевая ментальная модель Python: **переменная — это ярлык (ссылка) на объект**, а не коробка со значением.

```python
a = [1, 2, 3]
b = a            # b — ВТОРОЙ ярлык на ТОТ ЖЕ список
b.append(4)
print(a)         # [1, 2, 3, 4] — оба имени ссылаются на один объект
print(a is b)    # True
```

Присваивание **никогда не копирует** объект — оно лишь добавляет ссылку.

```python
x = 1000
y = x            # y ссылается на тот же int
y = 2000         # y ПЕРЕвязывается на новый объект; x не трогается
print(x)         # 1000
```

> Для immutable-объектов разница незаметна, для mutable — критична (см. билеты 21, 41).

## Итоги

- идентификатор: буквы/цифры/`_`, не с цифры, не ключевое слово, регистрозависим;
- `=` — это **связывание имени с объектом**, не копирование;
- форм связывания много: `=`, распаковка, `for`, `def`, `import`, `with`, включения…;
- переменная = ссылка (ярлык); присваивание копий не делает.
