# Билет 2: Python-скрипт — строки, комментарии, отступы

## Скрипт как текстовый файл

Python-программа — это обычный **текстовый файл** (обычно с расширением `.py`) в кодировке **UTF-8** (по умолчанию с Python 3). Интерпретатор читает его сверху вниз.

## Физические и логические строки

- **Физическая строка** — то, что заканчивается переводом строки в файле.
- **Логическая строка** — одна полная инструкция с точки зрения интерпретатора.

Обычно одна физическая строка = одна логическая. Но их можно рассинхронизировать:

```python
# Одна логическая строка на нескольких физических — явное продолжение через \
total = 1 + 2 + \
        3 + 4

# Неявное продолжение внутри скобок (), [], {} — \ не нужен
data = [
    1, 2, 3,
    4, 5, 6,
]

# Несколько логических строк на одной физической — через ;
x = 1; y = 2; z = 3     # допустимо, но не по PEP 8
```

> Предпочтительно неявное продолжение (скобки), а не `\`.

## Комментарии

```python
# Это комментарий — от # до конца физической строки
x = 5  # комментарий в конце строки
```

В Python **нет** синтаксиса блочных комментариев `/* ... */`. Многострочный текст в тройных кавычках — это **строковый литерал**, а не комментарий (хотя его используют как docstring):

```python
def f():
    """Это docstring — он доступен через f.__doc__."""
    pass
```

## Пустые строки и их мотивация

Пустые строки игнорируются интерпретатором, но важны для **читаемости**: PEP 8 рекомендует

- **2 пустые строки** между функциями/классами верхнего уровня;
- **1 пустую строку** между методами и для логического разделения блоков внутри функции.

```python
def first():
    pass


def second():       # 2 пустые строки до определения верхнего уровня
    a = compute()

    return a         # 1 пустая строка отделяет логический блок
```

## Отступы — часть синтаксиса

В Python **отступы заменяют фигурные скобки**: вложенность блока задаётся отступом, а не `{ }`.

```python
if x > 0:
    print("positive")      # тело if — отступ 4 пробела (PEP 8)
    print("still inside")
print("always")            # вне if — отступ снят
```

Правила:
- стандарт — **4 пробела** (не табы);
- **нельзя смешивать** табы и пробелы — `TabError`;
- неверный отступ — `IndentationError`.

```python
if True:
print("oops")      # IndentationError: expected an indented block
```

## Итоги

- логическая строка может занимать несколько физических (скобки или `\`);
- `;` позволяет несколько логических строк на одной физической;
- тройные кавычки — это **строка**, а не комментарий;
- пустые строки — только для читаемости (PEP 8: 2 между top-level, 1 внутри);
- отступы — **часть синтаксиса**, 4 пробела, без смешивания с табами.
