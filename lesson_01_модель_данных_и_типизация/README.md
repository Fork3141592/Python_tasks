## Задачи

### Задача 1.1. Псевдоним или копия

Список `original = [1, 2, 3]` уже создан. Реализуйте `make_pair(kind)`:
- при `kind == "alias"` вернуть пару `(original, second)`, где `second` — **то же
  самое** имя того же объекта (псевдоним);
- при `kind == "copy"` — где `second` **независимая** поверхностная копия.

Так, чтобы после `original.append(4)` псевдоним «увидел» изменение, а копия — нет.

### Задача 1.2. Только настоящие int

Верните из списка только элементы, которые являются **целыми числами, но не
булевыми**. Помните: `True` и `False` — экземпляры `bool`, а `bool` — подкласс
`int`, поэтому `isinstance(True, int)` истинно. Нужен более точный критерий.

`only_true_ints([1, True, 2, False, 3])` → `[1, 2, 3]`.

### Задача 1.3. Строгая типизация без падений

Python строго типизирован: `1 + "b"` не приводит типы молча, а бросает
`TypeError`. Реализуйте `safe_add(a, b)`: вернуть `a + b`, если операция
допустима, иначе `None`. Используйте подход EAFP (try/except), а не проверку
типов заранее — так функция заработает и для строк, и для списков.

### Задача 1.4. Предскажите вывод

**Не запуская код**, определите, что напечатает программа, и верни это строкой
из `predicted()`:

```python
x = [1, 2, 3]
y = x
def f(z):
    z = z + [4]
    return z
f(x)
x.append(99)
print(y)
```
