|**`ValueError`**|Тип правильный, но само **значение некорректно** для этой операции.|`int("привет")` _(тип строка, но внутри не числа)_|

как загрузить на проверку файлы
Чтобы загрузить измененный код в удаленный репозиторий (например, на GitHub, GitLab или Bitbucket) с комментарием, вам понадобится стандартная цепочка из трех-четырех команд в терминале.

Вот классический пошаговый процесс:

## Стандартный процесс (3 главных шага)

Перед выполнением команд откройте терминал в папке с вашим проектом.

### Шаг 1: Добавляем изменения в индекс (Stage)

Сначала нужно указать Git, какие именно измененные файлы мы хотим отправить. Команда ниже добавляет **все** измененные и новые файлы в текущей папке:

Bash

```
git add .
```

_(Если нужно добавить только один конкретный файл, вместо точки напишите его имя, например: `git add main.py`)_

### Шаг 2: Фиксируем изменения с комментарием (Commit)

Теперь создаем «слепок» кода (коммит) и пишем к нему комментарий. Флаг `-m` означает "message" (сообщение):

Bash

```
git commit -m "Исправил ошибку в цикле и добавил фильтрацию типов"
```

_Пишите комментарий внутри кавычек. Постарайтесь делать его коротким, но понятным._

### Шаг 3: Отправляем код в удаленный репозиторий (Push)

Выталкиваем ваши коммиты на сервер. Обычно главная ветка называется `main` или `master`:

Bash

```
git push origin main
```

_(Если вы работаете в другой ветке, например `develop`, то пишите `git push origin develop`)_

## Полезный лайфхак: Шаг 1 и Шаг 2 в одной команде

Если вы **не создавали новых файлов**, а только редактировали старые, вы можете объединить добавление и коммит в одну команду с помощью флага `-am`:

Bash

```
git commit -am "Быстрое исправление функции safe_add"
git push origin main
```

## Что делать, если Git ругается при `git push`?

Если сервер отклоняет ваш push, скорее всего, кто-то другой уже успел загрузить туда свой код, и ваша локальная версия устарела. В таком случае перед `git push` нужно скачать чужие обновления и объединить их со своими:

Bash

```
git pull origin main
```

И уже после того, как код успешно скачался и объединился, снова вводите:

Bash

```
git push origin main
```
