# Билет 9: Сопоставление с образцом (match-case)

## Что это

`match-case` (с Python **3.10**) — структурное **сопоставление с образцом** (structural pattern matching). Это не просто `switch`: образцы могут **разбирать структуру** объекта и связывать имена.

```python
def http_status(code):
    match code:
        case 200:
            return "OK"
        case 404:
            return "Not Found"
        case 500 | 502 | 503:      # несколько вариантов через |
            return "Server Error"
        case _:                    # wildcard — "всё остальное"
            return "Unknown"
```

`match` вычисляет subject (`code`) и проверяет `case` по порядку, выполняя **первый совпавший**.

## Мягкое ключевое слово _

`_` в `case` — **wildcard**: совпадает с чем угодно и **не связывает** имя. Это аналог `default` в switch. `_` — мягкое ключевое слово: вне `match` это обычное имя.

```python
case _:
    ...     # default-ветка
```

## Основные сценарии использования

### Литералы и «или»

```python
match command:
    case "start" | "go":
        ...
    case "stop":
        ...
```

### Захват значения (capture pattern)

```python
match point:
    case 0:
        print("ноль")
    case n:                    # связывает n с любым значением
        print(f"число {n}")
```

### Деструктуризация последовательностей

```python
match coords:
    case (0, 0):
        print("начало координат")
    case (x, 0):               # связывает x, требует y == 0
        print(f"на оси X: {x}")
    case (x, y):
        print(f"точка {x}, {y}")
    case [x, y, *rest]:        # x, y и остаток списка
        print(rest)
```

### Деструктуризация объектов и словарей, guard

```python
match obj:
    case {"type": "circle", "r": r}:       # совпадение по словарю
        area = 3.14 * r * r
    case Point(x=px, y=py):                 # совпадение по классу
        ...
    case n if n > 0:                        # guard — доп. условие
        print("положительное")
```

## Тонкость: имя vs значение

Одиночное имя в `case` — это **захват** (свяжет любое значение), а не сравнение со значением переменной. Чтобы сравнить со **значением константы**, используйте «точечный» паттерн:

```python
case Color.RED:        # сравнение со значением (через точку) — OK
    ...
case RED:              # ОШИБКА смысла: это захват, совпадёт с чем угодно!
    ...
```

## Итоги

- `match-case` с 3.10 — **структурное** сопоставление, мощнее `switch`;
- `case _` — wildcard/default (мягкое ключевое слово);
- умеет: литералы, `|`, захват имени, деструктуризацию (`(x, y)`, `[..]`, `{..}`, `Class(...)`), `guard` через `if`;
- одиночное имя в `case` = захват, не сравнение; для сравнения — `Class.ATTR`.
