# Билет 21: Поверхностная и глубокая копия

## Проблема: присваивание не копирует

`b = a` создаёт **второе имя** для того же объекта, а не копию:

```python
a = [1, 2, 3]
b = a
b.append(4)
print(a)        # [1, 2, 3, 4] — изменился и a!
print(a is b)   # True
```

## Поверхностная копия (shallow copy)

Создаёт **новый внешний объект**, но **вложенные объекты остаются общими** (копируются ссылки на них).

Способы сделать поверхностную копию списка:

```python
import copy
a = [1, 2, 3]
b = a[:]            # срез
b = list(a)         # конструктор
b = a.copy()        # метод
b = copy.copy(a)    # функция

print(b is a)       # False — внешний объект новый
```

Для плоского списка этого достаточно. Но при **вложенности**:

```python
a = [[1, 2], [3, 4]]
b = a.copy()             # поверхностная
b[0].append(99)          # меняем ВЛОЖЕННЫЙ список
print(a)                 # [[1, 2, 99], [3, 4]] — затронут и a!
print(a[0] is b[0])      # True — вложенные списки общие
```

## Глубокая копия (deep copy)

Рекурсивно копирует **все уровни вложенности** — полностью независимый объект:

```python
import copy
a = [[1, 2], [3, 4]]
b = copy.deepcopy(a)
b[0].append(99)
print(a)              # [[1, 2], [3, 4]] — не затронут ✅
print(a[0] is b[0])   # False — вложенные тоже разные
```

## Сводка

| Операция | Внешний объект | Вложенные |
|----------|----------------|-----------|
| `b = a` | тот же | те же |
| `a[:]`, `list(a)`, `a.copy()`, `copy.copy(a)` | **новый** | те же (общие) |
| `copy.deepcopy(a)` | **новый** | **новые** (рекурсивно) |

## Только для изменяемых это важно

У неизменяемых объектов (числа, строки, кортежи без вложенных mutable) разница незаметна — менять нечего. Проблема общих ссылок возникает с `list`, `dict`, `set` внутри.

(!) Кортеж тоже копируется поверхностно и может «протекать», если содержит изменяемые элементы:

```python
a = ([1, 2], 3)
b = copy.copy(a)
b[0].append(99)
print(a)        # ([1, 2, 99], 3) — кортеж неизменяем, но список внутри — нет!
```

## Итоги

**На экзамене:**
- `b = a` — не копия, а второе имя (`a is b`);
- **поверхностная** копия: `a[:]`, `list(a)`, `a.copy()`, `copy.copy` — новый внешний, общие вложенные;
- **глубокая**: `copy.deepcopy` — независима на всех уровнях;
- проблема общих ссылок — только у изменяемых вложенных объектов.
