# Билет 23: Кортежи как последовательности

## Все операции последовательностей

Кортеж поддерживает то же, что список, **кроме изменяющих** операций:

```python
t = (10, 20, 30, 40)

len(t)          # 4
t[1]            # 20      индексация
t[-1]           # 40
t[1:3]          # (20, 30) срез
t + (50,)       # конкатенация (новый кортеж)
t * 2           # повторение
for x in t: ... # итерация
min(t), max(t)  # 10, 40
sum(t)          # 100
sorted(t)       # [10,20,30,40] — но это СПИСОК, не кортеж!
```

(!) `sorted()` всегда возвращает **список**, даже для кортежа. Метода `.sort()` у кортежа нет (он бы менял на месте).

## Поиск элементов

Из методов у кортежа **только** `index()` и `count()` (неизменяющие):

```python
t = (10, 20, 30, 20)
print(20 in t)        # True
print(t.index(20))    # 1 — первое вхождение
print(t.index(20, 2)) # 3 — с позиции 2
print(t.count(20))    # 2
```

`index` без значения → `ValueError`:

```python
t.index(99)   # ValueError
```

## Особенности поиска

Как у списка: поиск **линейный O(n)**, сравнение по **значению** (`==`). Кортеж не даёт ускорения поиска — для O(1) нужен `set`/`dict`.

```python
nums = tuple(range(1000))
500 in nums       # O(n) — проходит элементы

# для частого поиска:
nums_set = set(nums)
500 in nums_set   # O(1)
```

## Чего у кортежа НЕТ

Поскольку кортеж неизменяем, отсутствуют: `append`, `extend`, `insert`, `remove`, `pop`, `clear`, `sort`, `reverse`, присваивание по индексу/срезу.

«Изменение» кортежа = создание нового:

```python
t = (1, 2, 3)
t = t + (4,)        # новый кортеж (1, 2, 3, 4)
t = t[:2] + (99,) + t[2:]   # «вставка»
```

## Итоги

- кортеж поддерживает все **неизменяющие** операции последовательностей;
- методы только `index()` и `count()`; `index` без значения → `ValueError`;
- поиск линейный O(n), по `==`; для O(1) — `set`/`dict`;
- `sorted(t)` → **список**; метода `.sort()` нет;
- «изменение» = создание нового кортежа.
