# Билет 19: Списки как последовательности

## Список — полноценная последовательность

К списку применимы все операции последовательностей (билет 16):

```python
s = [10, 20, 30, 40]

len(s)          # 4
s[1]            # 20    индексация
s[-1]           # 40
s[1:3]          # [20, 30] срез
s + [50]        # конкатенация (новый список)
s * 2           # повторение
[x for x in s]  # итерация
min(s), max(s)  # 10, 40
sum(s)          # 100
sorted(s)       # новый отсортированный
```

## Поиск элементов

### Оператор in

```python
print(20 in s)      # True
print(99 in s)      # False
print(99 not in s)  # True
```

### Методы index() и count()

```python
s = [10, 20, 30, 20]
print(s.index(20))      # 1 — индекс ПЕРВОГО вхождения
print(s.index(20, 2))   # 3 — поиск начиная с позиции 2
print(s.count(20))      # 2 — сколько раз встречается
```

`index` при отсутствии значения бросает `ValueError`:

```python
s.index(99)    # ValueError: 99 is not in list
```

Безопасный поиск:

```python
if 99 in s:
    idx = s.index(99)
```

## Особенность поиска: линейная сложность

Поиск в списке (`in`, `index`, `count`) — **линейный**, O(n): Python проходит элементы по очереди, сравнивая через `==`. Для частого поиска по принадлежности лучше `set` или `dict` (O(1), билеты 32, 37).

```python
# Медленно для больших данных:
if x in big_list:        # O(n)
    ...

# Быстро:
big_set = set(big_list)
if x in big_set:         # O(1)
    ...
```

## Поиск использует == , не is

`in`/`index` сравнивают по **значению** (`==`):

```python
print([1, 2, 3] in [[1, 2, 3]])   # True — равенство по значению
```

## Итоги

- к списку применимы все операции последовательностей;
- поиск: `in`, `index(value)`, `count(value)`;
- `index` без значения → `ValueError` (проверяйте через `in`);
- поиск в списке **линейный O(n)**, сравнение через `==`;
- для частого поиска — `set`/`dict` (O(1)).
