# Билет 15: Логические операторы not, and, or

## Операторы

```python
not x         # логическое отрицание → всегда bool
x and y       # И
x or y        # ИЛИ
```

`not` всегда возвращает `bool`:

```python
print(not True)    # False
print(not 0)       # True
print(not [1, 2])  # False
```

## Особенность: and/or возвращают ОПЕРАНД, а не bool

Это ключевая особенность Python. `and` и `or` **не приводят результат к bool** — они возвращают один из исходных операндов.

**`and`:** возвращает **первый ложный** операнд, иначе — последний:

```python
print(1 and 2)        # 2    (1 truthy → возвращает 2)
print(0 and 2)        # 0    (0 falsy → возвращает 0)
print("a" and "b")    # 'b'
print([] and 5)       # []
```

**`or`:** возвращает **первый истинный** операнд, иначе — последний:

```python
print(1 or 2)         # 1    (1 truthy → сразу возвращает 1)
print(0 or 2)         # 2
print("" or "default")# 'default'
print(None or 0)      # 0
```

## Ленивое вычисление (short-circuit)

`and`/`or` вычисляют правый операнд **только если нужно**:

```python
def side_effect():
    print("вызвана!")
    return True

False and side_effect()   # side_effect НЕ вызвана (and уже знает ответ)
True or side_effect()     # side_effect НЕ вызвана (or уже знает ответ)
```

Практическое применение — значения по умолчанию и защита от ошибок:

```python
name = user_input or "Аноним"        # если "" → "Аноним"
result = obj and obj.value           # не упадём, если obj is None
```

## Встроенные all() и any()

Обобщение `and`/`or` на итерируемые объекты:

```python
all([True, True, True])    # True  — все истинны
all([True, False])         # False
all([])                    # True  — на пустом всегда True!

any([False, False, True])  # True  — хоть один истинен
any([False, False])        # False
any([])                    # False — на пустом всегда False!
```

Удобно с генераторными выражениями:

```python
nums = [2, 4, 6]
print(all(n % 2 == 0 for n in nums))   # True — все чётные
print(any(n > 5 for n in nums))         # True — есть > 5
```

`all`/`any` тоже **ленивы**: останавливаются на первом решающем элементе.

## Итоги

- `not` → всегда `bool`; `and`/`or` → **возвращают операнд**, не bool;
- `and`: первый falsy или последний; `or`: первый truthy или последний;
- ленивость (short-circuit): правый операнд может не вычислиться;
- `x or default`, `obj and obj.attr` — типовые идиомы;
- `all([])==True`, `any([])==False`; обе ленивы.
