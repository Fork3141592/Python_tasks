
## Клонирование репозитория

**Если репозиторий ещё не склонирован на личный компьютер:**

```bash
git clone https://github.com/username/repo.git
```

Это создаст папку с проектом на вашем компьютере. Ссылку можно взять на GitHub, зайдя на репозиторий, который хотите скопировать.

---

**Если репозиторий уже есть локально и нужно подтянуть обновления:**

```bash
git pull
```

Это скачает изменения с удалённого репозитория и сразу смержит их в вашу текущую ветку.

Можно также сделать это поэтапно, чтобы понять какие изменения произошли:

```bash
git fetch          # скачать изменения без применения
git merge origin/main  # применить
```

---

**Если нужна конкретная ветка:**

```bash
git pull origin название-ветки
```

## Запрос логина и пароля при выполнении `git pull`

Если репозиторий подключён по HTTPS, а не по SSH, то при каждом выполнении `git pull`
командная строка будет запрашивать логин и пароль, что при частом использовании является очень неудобным. Есть два варианта разрешения данной проблемы:

---

**Вариант 1: Использовать SSH (рекомендую)**

1. Проверь, есть ли уже SSH-ключ:

```bash
ls ~/.ssh/id_ed25519.pub
```

Если ключ есть, команда `ls ~/.ssh/id_ed25519.pub` выдаст просто путь к файлу:

```
/home/ваше_имя/.ssh/id_ed25519.pub
```


2. Если нет — сгенерируйте:

```bash
ssh-keygen -t ed25519 -C "ваш@email.com"
```

3. Добавь публичный ключ на GitHub: **Settings → SSH and GPG keys → New SSH key**, вставьте содержимое файла `~/.ssh/id_ed25519.pub`
    
4. Смени URL репозитория с HTTPS на SSH:
    

```bash
git remote set-url origin git@github.com:username/repo.git
```
(Адрес можно взять если в репозитории войти в `Code` и перейти во вкладку `SSH`)

После этого `git pull` будет работать без пароля.

---

**Вариант 2: Использовать токен (быстрее настроить, но не рекомендую)**

Так как GitHub больше не принимает обычный пароль, то нужен **Personal Access Token**.

1. Создайте токен: **GitHub → Settings → Developer settings → Personal access tokens → Generate new token**
2. Выбери права `repo`
3. При запросе пароля вводите этот токен вместо пароля

Чтобы не вводить каждый раз, можно сохранить:

```bash
git config --global credential.helper store
```

После первого успешного `git pull` токен сохранится.

## Ветки и сброс изменений


**Просмотр названия веток**:

```bash
git branch -a
```

**Переключиться на ветку:**

```bash
git checkout <название_ветки>
```

---

**Сброс изменений**:

```bash
git restore .
```
